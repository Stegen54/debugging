Debugging HTML

“Everyone knows that debugging is twice as hard as writing a program in the first place. So if you're as clever as you can be when you write it, how will you ever debug it?”

— Brian Kernighan, The Elements of Programming Style

This HTML isn't doing what we want. Let's figure out why. Before jumping in, ask yourselves three questions:

  1. What do I expect this code to do?
  2. What does it actually do and is it different than what I expect?
  3. If it's different, how can I figure out what's wrong?
